# 	Chapter 01. 자료구조의 시작

# 1. 자료 구조란?

자료구조는 **컴퓨터에 자료를 효율적으로 저장하는 방식** 이다.

* **자료구조의 필요성과 목적**
  * 프로그램에서 사용하는 메모리를 절약하기 위해 (저장공간을 효율적으로 사용)
  * 프로그램 실행 시간을 단축하기 위해
  * 프로그램의 구현을 쉽게 하기 위해 



# 2. 자료구조의 분류

* 자료구조
  * **선형 구조**
    * 리스트(list)
    * 스택(stack)
    * 큐(queue)
  * **비선형 구조**
    * 트리(tree)
    * 히프(heap)
    * 그래프(graph)



## 2.1. 선형 구조

자료들 사이의 앞뒤 관계가 일대일(1:1) 이다. 이러한 구조는 단순하면서 동시에 직관적이어서 여러 프로그램에서 **자료를 차례대로 저장하는 목적** 으로 많이 사용한다.

**Ex) 리스트, 스택, 큐**



## 2.2. 비선형 구조

각각의 자료들 사이의 앞뒤 관계가 **일대일이 아니다**. **트리(tree)와 그래프(graph)** 등이 대표적인 비선형 구조이다. 

 이러한 비선형 구조는 현실 세계에서 실제 저장해야 하는 자료가 비선형 구조이기 때문에 필요하다. 예를 들어, 윈도우 탐색기에서 폴더 정보를 저장할 때 **계층 구조** 를 저장하기 위해 **트리가** 필요하다. 또한, 지도의 **도로 정보를** 저장할 때는 **그래프가** 필요하다.

 보통 **다대다 관계(many-to-many relationship)** 라고도 한다.



## 2.3. 그외: 단순 구조와 파일 구조

* **단순 구조** : 정수(int)나 실수(float  혹은 double), 문자(char) 및 문자열 등의 프로그램 언어에서 제공하는 기본적인 자료형(data type) 을 말한다.
* **파일 구조** : 보조기억 장치(ex. 하드디스크) 에 저장되는 파일의 자료구조이다. 주로 메모리에 한 번에 올릴(loading) 수 없는 대용량을 다룬다. **ex) 순차적(sequential) 파일 구조, 상대적(relative) 파일 구조, 색인(indexed sequential) 파일 구조, 다중키(multi-key) 파일 구조**
