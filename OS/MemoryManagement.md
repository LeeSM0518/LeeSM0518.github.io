메모리 관리

* 프로그램들은 실행하는 동안 **주메모리(RAM)** 에 올라와 있어야 한다.
* CPU 효율성을 위해 여러 프로세스들을 **동시에 메모리에 적재** 하고 있어야 한다.

# Background

배경

* 메모리는 각각 **주소가 할당된** 일련의 바이트들로 구성된다.



## 기본 하드웨어 (1)

**메인 메모리와** 프로세서 자체에 내장되어 있는 **레지스터들은** CPU가 직접 접근할 수 있는 유일한 **범용 저장 장치이다.**

모든 실행되는 명령어와 자료들은 메인 메모리와 레지스터에 있어야 한다.



## 기본 하드웨어 (2)

각각의 프로세스가 **독립된** 메모리 공간을 가지도록 보장해야 한다.

## 주소의 할당 (1)

프로그램이 실행되기 위해서는 **메인 메모리로 올라와서 "프로세스"가 되어야 한다**

* **입력 큐** : 디스크에서 메인 메모리로 들어오기를 기다리고 있는 프로세스들의 집합



## 주소의 할당 (2)

주소 바인딩 = 한 주소 공간에서 다른 주소 공간으로 사상시키는 것

* **메모리 공간에서의 명령과 자료의 바인딩**
  * 컴파일 시간 바인딩 => 절대 코드 (absolute code)
    * 컴파일 될 때 컴파일러에서 명령어들과 변수의 **메모리 주소를 결정**
  * 적재 시간 바인딩 => 재배치 가능 코드 (relocatable code)
    * 컴파일 된 프로그램 내의 각 변수나 명령어의 주소는 **특정 값을 기준으로 정해진다**
  * 실행 시간 바인딩
    * 매번 명령어가 실행될 때마다 **주소를 계산한다**

## 논리 대 물리 주소 공간 (1)

**논리 주소(logical address)** : CPU가 생성하는 주소

**물리 주소(physical address)** : 메모리에 나타나는 주소



* **바인딩 기법에 따른 주소**
  * <u>컴파일시/적재시</u> 주소 바인딩 기법의 경우에는 <u>논리, 물리 주소가 같다</u>
  * <u>실행시간</u> 주소 바인딩 기법의 경우에는 <u>논리, 물리 주소가 다르다</u>
* **메모리 관리기 (Memory Management Unit: MMU)**
  * 가상 주소에서 물리 주소로의 사상을 수행하는 하드웨어 장치
  * 재배치 레지스터 (relocation register) 사용

## 동적 적재

메모리에 적재되지 않고 재배치 가능한 적재 형태로 디스크에서 대기한다.

* 루틴이 필요한 경우에만 적재
* 동적 적재는 운영 체제로부터 특별한 지원을 필요로 하지 않는다

> 메모리 낭비를 막는다



## 동적 연결

**: 연결이 실행 시간까지 지연**

* 라이브러리 루틴을 바꿀 때 특히 유용하다 **(유지보수가 좋다)**
* 동적 연결과 공유 라이브러리는 **운영체제의 도움이 필요하다**

## 기본 스와핑

: 메인 메모리와 **예비 저장장치(Backing store, 하드 디스크)** 사이에서 프로세스를 이동시킨다.

* 프로세스를 스왑하려면 그 프로세스는 **완전히 휴지 상태** 에 있어야 한다
* 스와핑 시스템의 경우, **문맥 교환 시간** 이 상당히 오래 걸린다.
  * 스왑 시간의 대부분은 **디스크 전송 시간이다**



## Contiguous Memory Allocation

연속 메모리 할당 => 메인 메모리에 여러 프로세스를 수용하고 효율적으로 관리하기 위함

* 메모리는 일반적으로 두 부분으로 나누어진다
* **연속 메모리 할당** : 각 프로세스는 연속된 메모리 공간을 차지한다.

## 메모리 할당 (4)

**동적 메모리 할당 문제**

* **최초 적합 (first-fit)** : 프로세스 메모리보다 큰 공간들 중에서 <u>첫 번째 가용 공간에 할당</u>
* **최적 적합 (best-fit)** : 프로세스 메모리보다 큰 가용 공간들 중에서 <u>가장 작은 가용 공간을 할당</u>
* **최악 적합 (worst-fit)** : 프로세스 메모리보다 큰 가용 공간들 중에서 <u>가장 큰 가용 공간을 할당</u>



## 단편화 (1)

**외부 단편화 (external fragmentation)**

: 프로세스들이 메모리에 적재되고 제거될 때, 자유 공간은 작은 조각으로 나누어짐으로써 메모리에 **많은 수의 작은 자유 공간들이 존재한다.**



## 단편화 (2)

외부 단편화 문제 해결 => **아주 작은 가용 공간은 큰 요구에 포함시켜 할당**

* 작은 조각을 큰 공간에 포함하게 되는 **내부 단편화(internal fragmentation) 문제 발생 가능**



**밀집** 

* 메모리의 내용들을 적절하게 움직여서 모든 자유 공간을 하나의 큰 블록으로 만든다
* 재배치가 동적이고 수행시간에 이루어진다.
  * **메모리를 동적으로 바꿀때 시간이 오래걸려 사용X**



한 프로세스의 논리 주소 공간을 **여러 개의 비연속적인 공간으로 나누어 할당하는 방법**

: **세그먼테이션(논리적), 페이징(물리적)**

# Paging

* **외부 단편화** : 기억 장소가 연속되지 않고 산재된 많은 블록으로 흩어져 있을 때 발생한다.
* **페이징**
  * 논리 주소 공간이 연속적인 한 공간에 모여 있어야 한다는 **제약을 없앤다**
  * **하드웨어에 의존적**



## 기본 방법 (1)

메모리의 구성

* **물리 메모리** : <u>프레임(frame)이라</u> 불리는 고정 크기 블록으로 나누어진다
* **논리 메모리** : <u>페이지(page)</u> 라 불리는 <u>같은 크기의 블록으로</u> 나누어진다.

> 메모리 크기 : 프레임 = 메모리



## 기본 방법 (2)

**페이지 테이블 (page table)** : 각 페이지에 대해 실제 메모리에서의 시작 주소(base address)를 포함

* CPU에 의해 생성된 주소는 다음과 같이 둘로 나누어진다
  * **페이지 번호 (page number, p)** : 페이지 테이블의 index
  * **페이지 변위 (page offset, d)** : 해당 페이지에서의 데이터의 시작 주소

## 기본 방법 (7)

**페이징의 특징**

* 외부 단편화가 발생하지 않는다. 그러나 <u>내부 단편화는 발생</u> => 프레임 단위로 할당하므로
  * 페이지 사이즈 증가 = 내부 단편화 증가
* 동적 재배치의 형태 
  * 논리 주소는 페이징 하드웨어에 의해 물리 주소로 사상
* 운영체제에 의해 물리적 메모리의 할당 및 접근 관리
  * 논리 주소와 물리 주소가 다르다



## 하드웨어 지원 (1)

**페이지 테이블의 구현**

* <u>페이지 테이블을 메모리에 유지</u>

  * 메모리 위치에 접근하는데 많은 시간이 소요

  * 모든 데이터/명령에 대한 접근에 <u>두 번의 메모리 접근이 필요하다</u>

    > 페이지 테이블을 위해 한 번 + 데이터/명령을 위해 한 번

* <u>Translation look-aside buffers (TLBs: 변환 색인 버퍼)</u>

  * <u>매우 빠른 연관 메모리</u>

  * 프레임이 주소가 연관 메모리에 있으면 얻고, 없으면 페이지 테이블로부터 얻는다

# Structure of the Page Table

페이지 테이블 구조

* 페이지 테이블 구성 방법
  * 계층적 페이징 (Hierarchical paging)
  * 해시 페이지 테이블 (Hashed page tables)
  * 역 페이지 테이블 (Inverted page tables)



## 계층적 페이징 (1)

페이지 테이블을 **여러 개의 작은 조각으로** 나눈다

* **2단계 페이징 기법 (two-level paging scheme)**

  * 페이지 테이블의 페이지화

  * **32 bit 논리주소**

    * 페이지 번호 : 20 bit

    * 페이지 변위 : 12 bit

## 역 페이지 테이블 (1)

  * 메모리의 각 실제 페이제 대해 하나의 엔트리
  * 시스템에는 하나의 페이지 테이블만 존재

# Example

## IA-32 구조 (1)

**IA (Intel Architecture) - 32 시스템의** 메모리 관리는 세그먼테이션과 페이징의 두 부분으로 나눌 수 있다

* 논리 주소를 세그먼테이션 유닛에 전달
* 세크먼테이션 유닛은 논리 주소에 상응하는 선형 주소를 생성
* 선형 주소는 페이징 유닛에 전달되고 이어서 주기억장치의 물리 주소로 변환
