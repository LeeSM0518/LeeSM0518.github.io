# Disk Scheduling

디스크 스케줄링



디스크 드라이브에게 **빠른 접근 시간과 높은 전송량을** 제공해야 한다

* **디스크 접근 시간** = 탐색 시간 + 회전 지연 시간
  * **탐색 시간 (seek time)** : 헤드를 해당 실린더로 움직이는데 걸리는 시간
  * **회전 지연 시간 (rotation latency time)** : 헤드가 원하는 섹터 위치로 도달하기까지의 회전에 소요되는 시간
* **디스크 대역폭** : 단위 시간당 전송되는 총 바이트 수

<img src="../capture/스크린샷 2019-06-13 오후 10.15.30.png" width=500>



> 효율적인 스케줄링은 접근 시간과 대역폭을 모두 향상시킬 수 있다



## 선입 선처리 스케줄링

**FCFS Scheduling** : 디스크 스케줄링의 가장 간단한 형태. 큐에 들어온 순서대로 탐색

<img src="../capture/스크린샷 2019-06-13 오후 10.19.06.png" width=500>

> 헤드 이동 거리가 너무 길다



## 최소 탐색 시간 우선 스케줄링

**SSTF Scheduling** : 헤드가 먼 곳까지 이동하기 전에 현재 헤드 위치에 가까운 모든 요구를 먼저 처리한다.

<img src="../capture/스크린샷 2019-06-13 오후 10.21.24.png" width=500>

> 183 은 2번 째로 들어와도 매우 늦게 실행되고, 183 이 실행되기 전에 계속 프로세스가 추가된다면 183은 실행되지 않는다.



## SCAN 스케줄링

**SCAN Scheduling** : 입출력 헤드가 디스크의 한 끝과 다른 끝 사이를 계속해서 왕복하는 형태

<img src="../capture/스크린샷 2019-06-13 오후 10.23.48.png" width=500>



## C-SCAN 스케줄링

**C-SCAN Scheduling** : 대기 시간을 좀더 균등하게 하기 위해 헤드가 한쪽 끝에 도달하면 다시 처음부터 처리

<img src="../capture/스크린샷 2019-06-13 오후 10.25.20.png" width=500>



## LOOK 스케줄링

**LOOK Scheduling** : 현재 방향에서 더 이상의 요청이 없으면 헤드의 이동 방향을 바꾼다

<img src="../capture/스크린샷 2019-06-13 오후 10.26.45.png" width=500>



## 디스크 스케줄링 알고리즘의 선택

**최소-탐색-우선이 가장 일반적이고 자연스러운 선택**

* 디스크를 많이 사용하는 시스템의 경우에는 **SCAN 이나 C-SCAN 스케줄링이 적당하다**



특별한 이유가 없다면 **SSTF 나 LOOK이** 무난한 선택이 될 것이다
