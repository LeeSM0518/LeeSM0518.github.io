# File System Interface

파일 시스템 인터페이스



# File Concept

파일 개념

* **파일** : 보조 저장 장치에 기록된 관련된 정보의 집합
* **파일 시스템** : 관련된 정보 자료를 저장하는 실제적인 파일들의 집합체



## 파일 연산

**파일의 개방/폐쇄**

* 처음 사용시 파일을 개방(open)한다
* 파일이 더 이상 사용되지 않으면 폐쇄(close)한다



**오픈 파일 테이블 (open file table)**

* 운영 체제는 모든 **열린 파일에** 대한 정보를 가지는 작은 테이블을 유지



# Access Methods

**접근 방법** : 파일이 사용될 때 정보가 접근되어 컴퓨터에 메모리에 판독된다



## 순차 접근

파일에 있는 정보는 저장되어 있는 **레코드 순서로** 접근한다

* 컴파일러, 편집기
* 읽기(read) : 파일의 다음 부분을 읽은 후 자동으로 파일 포인터 증가
* 쓰기(write) : 파일의 끝에 첨가하고 새롭게 기록된 내용의 끝으로 포인터 이동

## 직접 접근

파일을 **특정한 순서 없이** 빠르게 레코드를 판독/기록하기 위해 **고정길이** 논리적 레코드로 구성

* 파일 = 연속된 블록이나 레코드의 집합



## 기타 접근 방법

**색인 접근 (ISAM: Indexed Sequential Access Method)**

* **색인 파일**
  * 색인은 여러 블록에 대한 **포인터를** 포함
  * 어떤 항목을 찾기 위해서는 먼저 **색인을 탐색하고 포인터를 사용하여** 파일을 직접 접근하고 원하는 항목을 찾아낸다.
