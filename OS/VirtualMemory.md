# Background

**가상 메모리 (Virtual memory)**

* 실제 물리 메모리 개념과 사용자의 논리 메모리 <u>개념을 분리</u>
* 프로세스의 전체가 완전히 메모리 내에 <u>존재하지 않아도</u> 수행이 가능한 기법

<img src="../capture/스크린샷 2019-06-12 오후 8.14.38.png">

> 메모리 사상 = page table
>
> 메모리 사이즈 = 물리 메모리 + 예비 기억 장치 (disk)



* **가상 주소 공간**
  * 프로세스가 메모리에 저장되는 논리적인 모습
  * 성긴(sparse) 주소 공간 **(invalid)**

## 요구 페이징의 성능 (1)

컴퓨터 시스템 성능에 중요한 영향을 미친다

* **유효 접근 시간 (effective access time: EAT)**

  <img src="../capture/스크린샷 2019-06-12 오후 8.38.27.png">

  > 페이지 폴트 확률 = 페이지 테이블에 없을 확률
  >
  > 페이지 폴트 시간 = 기본적인 오버헤드 + 페이지에서 나가는 시간 + 페이지에 올리는 시간 + 페이지 다시 시작 시간
  >
  > 요구 페이징 시스템에서 페이지 폴트 비율을 **낮게 유지시키는 것이** 상당히 중요

* **예 : 평균 페이지 폴트 서비스 시간이 8ms, 메모리 접근 시간이 200ns (시간 차이가 많이 난다)**

  <img src="../capture/스크린샷 2019-06-12 오후 8.43.13.png">

  * 유효 접근 시간은 <u>페이지 폴트 비율에 비례한다</u>

# Page Replacement

페이지 교체



<img src="../capture/스크린샷 2019-06-12 오후 9.58.14.png">

다중 프로그래밍의 정도가 높아짐에 따라 **과할당 발생**

* 자유 프레임 리스트에 비어 있는 프레임이 **없는 상황이 발생**



**해결책**

* 사용자 프로그램의 중단
* 프로세스의 swap out

=> <u>프로세스 교체 또는 페이지 교체(page replacement) 요구</u>
